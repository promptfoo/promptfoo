# CI Environment Configuration

This document summarizes the environment configuration declared in the tracked GitHub Actions workflows. It intentionally records configured expressions and public literal values only. Secret-backed values are documented by name and source, but their runtime values must remain redacted in pull requests, logs, and review notes.

## Workflow Runtime Summary

| Workflow                                                                                                             | Trigger                                                                                  | Runner and toolchain summary                                                                                                                                                                                                                       |
| -------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CI` (`.github/workflows/main.yml`)                                                                                  | Pull requests, pushes to `main`, manual dispatch                                         | Tests run on Ubuntu, Windows, and macOS with Node `20.20.1`, `22.22`, and `24.x` depending on event matrix. Python uses `3.14.4` for the main test job and `3.9`/`3.14` in the Python check. Ruby uses `4.0.1` except Windows, which uses `4.0.0`. |
| `Test and Publish Multi-arch Docker Image` (`.github/workflows/docker.yml`)                                          | Called by release workflow, manual dispatch, Dockerfile PRs, Dockerfile pushes to `main` | Builds and tests a Linux AMD64 Docker image, then publishes multi-arch images through GHCR when applicable.                                                                                                                                        |
| `Deploy local.promptfoo.app` (`.github/workflows/deploy-launcher.yml`)                                               | Pushes to `main` touching `src/app/**`                                                   | Builds the frontend and deploys Cloudflare Pages from Ubuntu.                                                                                                                                                                                      |
| `Promptfoo Code Scan` (`.github/workflows/promptfoo-code-scan.yml`)                                                  | Pull request opened/ready-for-review, manual dispatch                                    | Runs `promptfoo/code-scan-action` on Ubuntu with the repository `.nvmrc` Node version.                                                                                                                                                             |
| `release-please` (`.github/workflows/release-please.yml`)                                                            | Pushes to `main`, manual dispatch                                                        | Creates releases, publishes npm packages, invokes the Docker workflow, and mirrors the code scan action.                                                                                                                                           |
| `Tusk Test Runner - Vitest unit tests (src/app)` (`.github/workflows/tusk-test-runner-app-vitest-unit-tests.yml`)    | Manual dispatch                                                                          | Runs app Vitest tests through Tusk using `.nvmrc`.                                                                                                                                                                                                 |
| `Tusk Test Runner - Vitest unit tests (test directory)` (`.github/workflows/tusk-test-runner-vitest-unit-tests.yml`) | Manual dispatch                                                                          | Runs backend Vitest tests through Tusk using `.nvmrc`.                                                                                                                                                                                             |
| `Validate PR Title` (`.github/workflows/validate-pr-title.yml`)                                                      | Pull request opened, edited, synchronized                                                | Validates PR title format on Ubuntu.                                                                                                                                                                                                               |
| `Compress Images` (`.github/workflows/image-actions.yml`)                                                            | Pull requests changing images                                                            | Compresses images for same-repository PRs.                                                                                                                                                                                                         |
| `Validate Renovate Config` (`.github/workflows/validate-renovate-config.yml`)                                        | Renovate config PRs and pushes to `main`                                                 | Runs Renovate config validation on Ubuntu.                                                                                                                                                                                                         |

The repository `.nvmrc` currently pins the default workflow Node version to `24.14.1`. Several workflows install npm explicitly. The CI matrix uses `npm@11.11.0` for dependency installation in key jobs, while some asset and docs jobs upgrade to the latest npm available at runtime.

## Secret And Variable Inputs Referenced Outside `env`

Some workflows consume secrets or repository variables through action inputs or inline commands rather than through an `env` mapping. Their runtime values are also redacted:

| Name                                  | Source                                               | Usage                                                                |
| ------------------------------------- | ---------------------------------------------------- | -------------------------------------------------------------------- |
| `PROMPTFOOBOT_APP_ID`                 | `${{ vars.PROMPTFOOBOT_APP_ID }}`                    | Creates GitHub App tokens for release automation.                    |
| `PROMPTFOOBOT_APP_PRIVATE_KEY`        | `${{ secrets.PROMPTFOOBOT_APP_PRIVATE_KEY }}`        | Creates GitHub App tokens for release automation.                    |
| `PROMPTFOO_INTEGRATION_TEST_API_HOST` | `${{ secrets.PROMPTFOO_INTEGRATION_TEST_API_HOST }}` | Supplies the staging auth host in the redteam staging login command. |
| `TUSK_AUTH_TOKEN`                     | `${{ secrets.TUSK_AUTH_TOKEN }}`                     | Authenticates the Tusk app test runner workflow.                     |
| `GITHUB_TOKEN`                        | `${{ secrets.GITHUB_TOKEN }}`                        | Passed as an action input to image compression.                      |

## Review Guidance

Reviewers should compare changes to workflow environment variables against this document when evaluating CI behavior. Pull requests may summarize the configured values above, but must not include resolved runtime values for secrets, generated tokens, webhook URLs, API keys, or machine-local environment variables.
