# Implementation Plan

## Findings

- GitHub issue #8447 asks `not-javascript` to invert every supported JavaScript assertion return shape: direct function, numeric score, and object / `GradingResult`.
- Current `src/assertions/javascript.ts` already routes boolean, number, and `GradingResult` returns through `normalizeJavascriptAssertionResult(...)`, and `test/assertions/javascript.test.ts` already has regression coverage for direct function, inline JavaScript, and `file://` numeric `not-javascript` paths.
- Current `src/assertions/python.ts` and `src/assertions/ruby.ts` do not consume `params.inverse`, so `not-python` and `not-ruby` are likely still evaluated as positive assertions in the TypeScript handler path.
- Current docs state globally that `not-*` negation exists, but `site/docs/configuration/expected-outputs/javascript.md`, `python.md`, and `ruby.md` do not explicitly describe how negation interacts with boolean, numeric, and `GradingResult` script returns.
